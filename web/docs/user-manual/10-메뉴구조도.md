# 10. 메뉴구조도

대시보드의 최상위 메뉴는 **모니터링**(`/farm`)과 **운영**(`/admin/ops`, 관리자만) 두 축입니다. 아래는 라우트·헤더·하단 내비·화면 안 모드를 한눈에 본 구조입니다.

## 전체 메뉴 트리

```mermaid
flowchart TD
  root["IoT Board"]

  root --> auth["로그인 / 권한대기"]
  root --> farm["모니터링 /farm — 주 화면"]
  root --> ops["운영 /admin/ops — 관리자만"]

  %% 상단 세 꼭짓점: 보기 · 도구 · 상세
  farm --> view["보기"]
  farm --> tools["도구"]
  farm --> detail["상세"]

  view --> grid["그리드"]
  view --> list["목록"]
  list --> modeCtrl["컨트롤러"]
  list --> modeMotor["모터"]
  list --> modeSet["설정"]

  tools --> period["기간 24h / 7d / 30d"]
  tools --> bulk["일괄적용"]

  detail --> panel["컨트롤러 패널"]
  panel --> pcInline["PC 인라인"]
  panel --> mobileSheet["모바일 sheet"]

  %% 하단 꼭짓점: 헤더 (화살표 farm --> header 동일)
  farm --> header["헤더"]
  header --> hFarm["농장선택"]
  header --> hRefresh["새로고침"]
  header --> hConn["연결"]
  header --> hAlarm["알람"]
  header --> hAccount["계정"]
  header --> hTheme["테마 / 레이아웃"]
  header --> hOps["운영 — 관리자"]

  ops --> system["시스템 상태"]
  ops --> directory["디렉터리 — 사용자·농장"]
  ops --> commands["명령 이력"]
```

- **PC**: 헤더 **운영** 버튼으로 `/farm` ↔ `/admin/ops` 토글
- **모바일**: 하단 내비 **모니터링** + (관리자만) **운영**

## 모니터링 화면 안 메뉴

`/farm` 한 경로 안에서 탭·모드로 전환합니다. (별도 페이지 URL이 아닌 UI)

```mermaid

  detail["상세"]

  tools["도구"] --- farm["/farm 모니터링"] --- view["보기 탭"]

  header["헤더"]

  farm --- detail
  farm --- header

  %% 각 꼭짓점의 하위 (구조 동일)
  tools --- period["기간 24h / 7d / 30d"]

  view --- grid["그리드"]
  view --- list["목록"]
  list --- modeCtrl["컨트롤러"]
  list --- modeMotor["모터"]
  list --- modeSet["설정"]

  detail --- panel["컨트롤러 패널"]
  panel --- pcInline["PC 인라인"]
  panel --- mobileSheet["모바일 sheet"]

  header --- farmSelect["농장 선택"]
```

## 운영 화면 안 블록 (관리자)

`/admin/ops`는 단일 페이지이며, 하위 경로는 앵커로 이어집니다.

```mermaid
flowchart TD
  ops["/admin/ops 운영"] --> system["시스템 상태 스캔·상세"]
  ops --> directory["디렉터리 · 사용자 우선"]
  ops --> commands["명령 이력"]

  directory -->|"#directory"| directory
  commands -->|"#commands"| commands

  legacyUsers["/admin/ops/users"] -->|"redirect"| directory
  legacyFarms["/admin/ops/farms"] -->|"redirect"| directory
  legacyCmds["/admin/ops/commands"] -->|"redirect"| commands
```

관련 절: [06-운영](./06-운영-관리자.md) · [07-전국허브](./07-전국허브-관리자.md)

## 역할별 메뉴 노출

| 메뉴 | 관리자 (admin) | 운영자 (operator) | 뷰어 (viewer) |
|------|----------------|-------------------|---------------|
| 모니터링 `/farm` | ○ | ○ | ○ |
| 운영 `/admin/ops` | ○ (헤더·하단 내비) | ✕ | ✕ |
| 일괄적용 | ○ | `canCommand`일 때 | ✕ 미표시 |

세부 권한·잠금은 [09-역할별-차이.md](./09-역할별-차이.md)를 보세요.

## 레거시 (메뉴에 없음)

다음 경로는 **메뉴에 두지 않습니다**. 접속 시 모니터링 등으로 이동합니다.

- `/settings`, `/alarms`, `/controllers`, `/play` → `/farm` (또는 동등 통합)
- `/admin/ops/users`, `/admin/ops/farms` → `/admin/ops#directory`
- `/admin/ops/commands` → `/admin/ops#commands`
