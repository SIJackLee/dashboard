# 9. 역할별 차이 요약

권한의 핵심은 **조회(`can_read`)** 와 **명령(`can_command` / `canCommand`)** 입니다. 관리자는 전역 관리 + 명령이 가능하고, 운영자는 배정 농장에서 명령, 뷰어는 조회만 합니다. 메뉴 노출 트리는 [10-메뉴구조도.md](./10-메뉴구조도.md)를 보세요.

## 한눈에 보기

| 기능 | 관리자 (admin) | 운영자 (operator) | 뷰어 (viewer) |
|------|----------------|-------------------|---------------|
| 모니터링 그리드·목록 | ○ | ○ (배정 농장) | ○ (배정 농장) |
| 히트맵·그래프 조회 | ○ | ○ | ○ |
| 단건 설정 **적용** | ○ | ○ (`can_command`) | ✕ 조회 전용 |
| 일괄적용 | ○ | ○ | ✕ 미표시 |
| 헤더 **운영** → `/admin/ops` | ○ | ✕ | ✕ |
| 전국 허브·다농장 전환 | ○ | 제한적(배정분) | 제한적(배정분) |
| 계정 「기능 안내」 투어 | ○ | ○ | ○ |

## 화면별 체감

### 설정 패널

- **운영자·관리자**: 슬라이더 변경 → **적용** 활성 → 전송·ACK.
- **뷰어**: 「조회 전용」 배너, 슬라이더·적용·기본값 잠금.

### 일괄적용

- 명령 권한이 없으면 토글이 보이지 않습니다.

### 운영·허브

- **관리자만** 헤더에 **운영**이 보입니다.
- `/admin/ops`에 비관리자가 들어가면 모니터링으로 복귀합니다.

## 권한 기준 (참고)

앱 내부에서 명령 가능 여부는 대략 다음입니다.

- 관리자이거나
- 사용자 접근 목록에 `can_command`가 하나 이상 있는 경우 → `canCommand === true`

세부 농장 부여는 관리자 운영 화면의 디렉터리에서 조정합니다. → [06-운영-관리자.md](./06-운영-관리자.md)
